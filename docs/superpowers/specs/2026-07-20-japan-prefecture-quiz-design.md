# 都道府県クイズアプリ 設計ドキュメント

作成日: 2026-07-20
対象ユーザー: 小学4年生（日本の都道府県の名前と位置を覚える）

## 1. 概要

ブラウザで遊べる、都道府県の名前と場所を覚えるためのクイズアプリ。
**単一のHTMLファイル**で完結し、ダブルクリックで開けばそのまま遊べる（サーバー不要・オフライン動作）。
デザインは**ゲーム風**（明るい配色、スコア表示、派手なエフェクト）。

1トライごとにタイムと正答率を計測し、「全問正解の最速」を目指す。記録はブラウザ（localStorage）に保存する。
正解のたびに、その都道府県に関係の深い絵文字が画面に降ってきてバウンドする。**回答が速いほど大量に**降る。

## 2. ゲームの構成

3つの軸を組み合わせて出題する。

### 2.1 スコープ（出題の見せ方）
- **通常モード**: 日本地図全体を表示して出題する
- **単独モード**: 都道府県の形（シルエット）だけを1つ表示して出題する
- **サッカーモード**: Jリーグのチーム名を見て、本拠地の都道府県を地図で答える（独立した専用モード）

### 2.2 方向
- **場所→名前**: 場所（地図でハイライト／形）を見て、名前を答える
- **名前→場所**: 名前を見て、場所（地図／形）を答える

### 2.3 レベル
- **イージー**: 3択から選ぶ
- **ハード**: 漢字でタイプして答える

### 2.4 組み合わせ表（確定）

| スコープ | 方向 | イージー(3択) | ハード(タイプ) |
|---|---|---|---|
| 通常 | 場所→名前 | 地図で1県が光る → 3択の名前ボタンから選ぶ | 光った県の名前を漢字でタイプ |
| 通常 | 名前→場所 | 名前を表示 → 地図上で「正解を含む3県」が光り、その中からクリック | 名前を表示 → 地図の県を直接クリック（光るヒントなし） |
| 単独 | 場所→名前 | 県の形を表示 → 3択の名前ボタンから選ぶ | 県の形を表示 → 名前を漢字でタイプ |
| 単独 | 名前→場所 | 名前を表示 → 3つの形から正しい形をクリック | （形はタイプできないため無効。この組み合わせは選択不可にする） |
| サッカー | チーム→場所 | チーム名を表示 → 地図の県をクリック | 同左（サッカーはレベル分けなし、クリック方式） |

- 「単独 × 名前→場所 × ハード」は成立しないため、メニュー上で選べないようにする（グレーアウト）。

### 2.5 1トライの流れ
- 対象となる都道府県すべてを1問ずつ出題する（順番はシャッフル）。
  - 通常/単独モード: 47都道府県すべて。
  - サッカーモード: Jリーグチームが本拠地を置く都道府県のみ（下記データ参照）。
- 全問終わったらリザルト画面で「合計タイム」「正答率（正解数/問題数）」「ミス数」を表示。
- **全問正解を達成したトライのみ**、合計タイムをそのモード設定のベスト記録として保存・更新する。
- 回答済みの県は、通常モードの地図上で**その県の代表絵文字マーク付き**で塗られ、進捗が一目で分かる。

## 3. 地図データ

「形だけで答える単独モード」があるため、**本物の都道府県シルエット**が必須。

### 3.1 データソース
- `https://github.com/dataofjapan/land` の `japan.geojson`（パブリック、47都道府県、JISコード1〜47、日本語名 `nam_ja` 付き、MultiPolygon）を使用。
- 元データは約13MB・約8万点と巨大。

### 3.2 変換パイプライン（ビルド時、実装者が実行）
1. GeoJSONを取得。
2. ポリゴンを簡略化（mapshaper の Visvalingam 法などで 3〜5% 程度まで間引き）。細かい離島は落とすが、都道府県の識別に必要な主要な島（沖縄本島、佐渡、淡路島等）は残す。
3. 緯度経度を等角風の単純投影（メルカトル近似、もしくは経度×cos(緯度中央値)）でSVG座標に変換し、viewBoxに収める。
4. 都道府県ごとに `<path d="...">` 文字列を生成。座標は小数1〜2桁に丸めてサイズ圧縮。
5. あわせて各県の**重心（ラベル・マーク配置用）**と**バウンディングボックス**を計算。
6. 目標: 埋め込み後の地図データ合計で概ね 150〜400KB 以内。
7. 生成物を単一HTML内の JavaScript データ（`PREF_PATHS[id] = "..."` 等）として埋め込む。

### 3.3 表示
- **通常モード**: 全都道府県のパスを1つのSVGに描画。出題対象の県をハイライト、回答済みの県を絵文字マーク付きで塗る。クリック/タップで county を選択。
- **単独モード**: 対象県のパスだけを、そのbboxに合わせて拡大表示（向き・比率は保持）。

## 4. データモデル

各都道府県（JISコード順）に以下を持つ静的データを用意する。

```
Prefecture {
  id:        number   // JISコード 1..47
  kanji:     string   // 例 "北海道"
  kana:      string   // 例 "ほっかいどう"（読み・ハード判定の補助）
  accept:    string[] // タイプ回答で正解と見なす表記ゆれ（例 ["北海道"]、"東京"/"東京都" 等）
  emojis:    string[] // その県に関係の深い絵文字（3〜6個）例 北海道:["🐄","🦀","⛄","🐻","🍜"]
  teams:     string[] // Jリーグ（J1〜J3）チーム名。無い県は []
  path:      string   // SVGパス（3.2で生成）
  centroid:  [x,y]    // ラベル/マーク位置
  bbox:      [x,y,w,h]// 単独モードの拡大用
}
```

- **絵文字**: 47県ぶんを手作業で用意（名産・名所・シンボルなど、小4が「なるほど」と思えるもの）。
- **Jリーグ**: チーム→本拠地の県の対応表を用意。1県に複数チームがある場合は全て正解チームとして扱い、出題は「県ごとに1問（その県のチームのどれか1つを提示、複数あればランダム）」または「チームごとに1問」。→ **チームごとに1問**とし、提示したチームの本拠地の県をクリックさせる。タイプ回答時の表記ゆれに配慮した `accept` 相当の別名も持たせる。

## 5. 画面構成

1. **タイトル / メニュー画面**
   - モード（通常/単独/サッカー）、方向（場所→名前/名前→場所）、レベル（イージー/ハード）を選択。成立しない組み合わせは無効化。
   - 各設定のベスト記録（あれば）を表示。「スタート」ボタン。
2. **クイズ画面**
   - 上部: 経過タイム、進捗（例 12/47）、現在の正答/ミス数。
   - 中央: 地図 または 形 または チーム名。
   - 回答UI: 3択ボタン / タイプ入力欄 / 地図クリック。
   - 正誤フィードバック（正解=絵文字が降る、不正解=軽い揺れ＋正解を短く提示）。
3. **リザルト画面**
   - 合計タイム、正答率、ミス数、ベスト更新の有無。
   - 「もう一度」「メニューへ」。

## 6. 演出：絵文字レイン

- 正解時、その県の `emojis` からランダムに選んだ絵文字を多数生成して上から落とす。
- **数は回答の速さに反比例**（速いほど多い）。例: `count = clamp(round(60 - 経過秒*8), 12, 80)` のような式で、速答ほど多く、遅いと最低数。
- 物理: 各絵文字に初速・重力・反発係数を持たせ、画面下端や左右の壁で**バウンド**。数秒かけて減衰し消える。
- 実装は軽量な自前アニメーション（`requestAnimationFrame`）。パフォーマンスのため上限（同時最大数）を設ける。DOM要素 or Canvas のどちらか実装しやすい方（DOMのspanで実装、上限管理）。

## 7. 記録（localStorage）

```
key: "jp-quiz-records-v1"
value: {
  [settingKey]: { bestTimeMs: number, playedCount: number, lastPlayedISO: string }
}
settingKey 例: "normal|place2name|easy", "solo|name2place|easy", "soccer"
```
- 全問正解トライのみ `bestTimeMs` を更新（より速ければ）。

## 8. 技術方針

- 単一 `index.html`。外部ライブラリ・外部フォント・外部画像に依存しない（絵文字はシステム標準）。
- Vanilla JS + SVG。ビルド工程は「GeoJSON→パス埋め込み」の一度きりで、成果物HTMLは自己完結。
- レスポンシブ（PC/タブレット想定、クリック=タップ両対応）。

## 9. テスト方針

ロジック部分（純粋関数）を中心に検証する。UIは手動確認。
- **絵文字レインの個数計算** `emojiCountForTime(sec)`: 速い→多い、遅い→少ない、上限下限を守る。
- **タイプ回答の正誤判定** `judgeTyped(input, pref)`: 表記ゆれ（"東京"/"東京都"、前後空白）を吸収。
- **3択の生成** `makeChoices(answer, all)`: 正解を必ず1つ含み重複しない3件。
- **通常×名前→場所×イージーのヒント3県** `makeMapHints(answer, all)`: 正解を含む相異なる3県。
- **記録更新** `updateRecord(records, key, timeMs, allCorrect)`: 全問正解時のみ、より速ければ更新。
- **地図データの健全性**: 47県すべてに path/centroid/bbox/emojis が存在し、パスがパース可能。
- 小さなテストHTML（`tests.html`）にアサーションを書き、コンソールで pass/fail を確認する方式。

## 10. スコープ外（YAGNI）

- オンラインランキング・アカウント・通信同期。
- 音声・BGM（初版では入れない。必要なら後日）。
- 市区町村レベルの出題。
- 地方別・難易度カスタム等の追加フィルタ。
