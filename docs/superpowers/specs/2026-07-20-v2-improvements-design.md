# 都道府県クイズ v2 改善 設計ドキュメント

作成日: 2026-07-20
対象: 既存の都道府県クイズアプリ（`inutano/todofuken-quiz`）への改善

## 概要

ユーザー要望のうち、次の**3点**を実装する。

1. **ゆめかわパステルデザイン**への刷新（見た目のみ）
2. **名産絵文字レインの改善**（底に積もってしばらく残ってから消える）
3. **【バグ修正】** 3択/地図ヒントに回答済みが混じり実質1〜2択になる問題の解消

### 見送り: ゆるキャラモード
当初検討した「ご当地ゆるキャラモード（Wikimedia画像使用）」は**見送る**。理由: Wikimedia Commons の自由ライセンス画像は、ゆるキャラ本体のきれいな写真がほとんど無く（キャラ意匠が著作物のため）、残っているのは主にラッピング電車・飛行機・グッズ・イベント遠景で、自動取得では別物（プリン・教会・政治家の写真など）を掴んでしまう。キッズ向けクイズに耐える正しい画像は10県前後・品質もバラつくため、実装しない（実地検証済み）。

既存アーキテクチャ（`src/data.js` / `logic.js` / `map.js` / `rain.js` / `app.js` / `styles.css`、`index.html`、`build/inline.py` で単一 `japan-quiz.html` を生成）を踏襲。単一HTML・外部リソースなし・オフライン動作の原則を維持する。

## 1. ゆめかわパステルデザイン

`src/styles.css` の配色・装飾を全面刷新する（DOM構造・ロジック・クラス名は変更しない）。

- **配色（CSS変数を差し替え）**:
  - 背景: 淡いラベンダー→ミントのやわらかグラデ（例 `#f6e6ff` → `#e6f7ff`）
  - カード: ほぼ白＋うっすらピンクの縁、丸み大（border-radius 26px）、ふんわり影
  - アクセント/ボタン: パステルピンク `#ffd1e8`、文字は可読性確保のため濃いめのパステル紫 `#7a5aa6`、ボタン影も同系のくすみピンク
  - 正解=ミント `#b8f0d0`、不正解=コーラル `#ffc2cf`
  - 地図の県: 通常=パステルミント `#bfead0`、ターゲット=パステルコーラル `#ffb0a8`、ヒント=パステルピンク `#ffc7e0`、回答済み=パステルイエロー `#fff0b0`
- **装飾**: タイトルに♡・✧の軽い装飾（絵文字またはCSS疑似要素）、ボタンは丸ゴシック太字＋やわらかい影（押すと沈む既存演出は維持）。
- **フォント**: 既存の丸ゴシック系スタック継続（`Hiragino Maru Gothic ProN` 等、外部フォント不使用）。
- 変更は色・角丸・影・装飾に限定。レイアウト構造（screen/hud/stage/answer）とセレクタは不変。
- 注意: `map.js` は塗り色に CSS 変数（`--pref` 等）を参照しているため、CSS変数名を維持すれば地図色も自動追従する。**CSS変数名は変えない**（値のみ変える）。

## 2. 名産絵文字レインの改善

`src/rain.js` を改修し、「降る→バウンド→**底に積もって静止ししばらく残る→フェード**」の挙動にする。

- パーティクルに静止状態 `resting` と静止経過 `restT` を持たせる2段階化:
  - (a) 落下・バウンド期: 既存の重力＋反発。
  - (b) 静止期: 床付近で速度が十分小さくなったら着地させ（`resting=true`、`y=FLOOR`、`vx=vy=0`）その場に留める。
- **寿命を延長**（例 `life` を 3.5〜4.5 秒程度に）。着地後もほぼ不透明で保持し、寿命末尾の約0.8秒でフェードアウトして消す。
- 「速いほど多い」個数計算（`emojiCountForTime`）は不変。同時最大数 `MAX` は積み上がりを見て必要なら微増（例 120→160）。
- 実装は既存の `requestAnimationFrame` ループ内で完結。`burst(emojis, count)` の外部インターフェースは不変。

## 3. 【バグ修正】回答済みを選択肢から除外

**原因**: `makeMapHints`（通常×名前→場所×イージー）と `makeChoices` のダミー候補が全県からランダムのため、回答済み県が混じる。特に地図ヒントでは、`setStates` 後に走る `reapplyAnswered()` が回答済み県を黄色（answered）で上書きするため、ヒント色（ピンク）に見えるのが実質1〜2県になり「選択肢が減る」ように見える。

**修正**:
- `logic.js` の `makeChoices(answerId, allIds, rand)` と `makeMapHints(answerId, allIds, rand)` に **除外集合 `exclude`** を追加し、`makeChoices(answerId, allIds, rand, exclude = [])` / `makeMapHints(answerId, allIds, rand, exclude = [])` とする。ダミーは `allIds − {answer} − exclude` から選ぶ。
- **フォールバック**: 除外後のプールが不足（必要数=3 に満たない）場合は、`exclude` の要素も使って不足分を補い、**常に3件（正解を必ず含む・重複なし）** を返す（終盤で未回答が少ないケース）。`exclude` に `answerId` が含まれていても、返り値には必ず `answerId` を1つだけ含める。
- `app.js` の呼び出し側で、現在の回答済み集合（`answeredMarks` のキー配列を数値化したもの）を `exclude` として渡す。
- 適用箇所（`app.js`）:
  - 3択の場所→名前（`nameChoiceButtons`）
  - 単独×名前→場所の3形選択（`renderSoloName2Place`）
  - 通常×名前→場所イージーの地図ヒント（`renderNormalName2Place` → `makeMapHints`）
- サッカー等の他モードのロジックは不変。

## テスト方針（`build/test.js` に追加）

- `makeChoices` with `exclude`: 正解を必ず含む／`exclude` の県が結果に出ない／件数3・重複なし。
- `makeChoices` フォールバック: 未回答が少なく `allIds − {answer} − exclude` が2件未満でも、常に3件（正解含む）を返す。
- `makeMapHints` with `exclude`: 同上（正解含む3件・除外反映・フォールバック）。
- 既存テスト（`makeChoices`/`makeMapHints` の3引数呼び出し）が後方互換で通ること（デフォルト `exclude=[]`）。
- パステルデザイン・絵文字の積もる挙動は、ヘッドレスブラウザのスクリーンショットで目視確認（コントローラ実施、ユニットテスト対象外）。

## スコープ外（YAGNI）

- ゆるキャラモード（上記理由で見送り）。
- デザインテーマ切替（パステル固定）。
- 絵文字の物理を精密化する等の作り込み（積もって残る体験が満たせれば十分）。

## 既知の割り切り

- 絵文字が積もって残る間、次の正解のレインと重なりうるが、`MAX` 上限と寿命フェードで自然に解消する範囲とする。
- デザイン変更は値のみ。DOM・クラス・CSS変数名を変えないため、`map.js` の色参照や既存の目視検証結果を壊さない。
